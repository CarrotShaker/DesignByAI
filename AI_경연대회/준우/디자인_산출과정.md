# 산출과정

## 디자인 프롬프트 ( md ) 작성

1. Claude desktop ai를 활용하여 디자인 프롬프트를 먼저 요청 
2. md 파일을 보면서 일부 수정 과정을 거침
    - token 시스템을 위해서 색상 / 텍스트 토큰등은 따로 분리해줘
    - 전체화면 뿐만 아니라 상세화면에 대한 디자인도 해줘
3. Antigravity 를 통해서 부족한 점 교차검증을 위해 md 파일 재검토
    - 제안사항에 따라 말줄임처리 ( Ellipsis ) 및 Ripple 효과 를 보충

## 디자인 AI 선정

리서치 결과 프롬프트 기반 구현에 가장 적합하다고 판단되는 다음 세가지 AI를 선정

- https://stitch.withgoogle.com/

- https://readdy.ai/

- https://www.figma.com/make/

### Stitch

구글에서 지원하는 gemini 기반 ui 생성 ai 

장점 

- md 파일을 직접 업로드 가능
- 화면별 스펙을 각각 확인 및 직접 화면별로 편집도 지원

단점

- 영어로 나온다..?
- 아무래도 stitch 에서 figma 로 변환해서 옮기다보니 프로토타입을 통해 인터랙션 하려면 추가 과정이 필요하다.

## Figma make

장점

- figma 기반이라 가장 figam에 적잡한 형태

단점

- 처음 산출물이 와이어프레임 형태가 아닌 figma preview 형태로 나옴
- figma 토큰이 너무너무 빨리 닳음…
- figma desgin으로 변환하려면 요금제 결제 필요

## Readdy

장점 

- 구현과 동시에 figma preview 처럼 인터렉션까지 가능하도록 산출물이 나옴

단점

- 직접 md 업로드 지원하지 않음
- 글자수 제한 5000자
- 코드스펙 및 figma 를 export 하는게 유료
