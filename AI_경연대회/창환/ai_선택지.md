## 1. 디자인 ai 선택지
구글링에서는 너무많은 ai 툴이 광고로 노출되어 오히려 적절한 툴을 찾을수 없어 유튜브 영상 여러개를 시청해서
대략적인 사용형태를 확인하는 형태로 진행하였다.

하기 영상이 제일 영양가 있어서 해당 영상을 기준으로 툴을 정리해보았다.
해당 영상은 제작자는 기획 혹은 디자이너분 같은데 해당 관점에서 툴을 사용하는 예시를 잘보여준다.

번외: 영상에서 보면 서버 API 구성을 위해서 supabase key를 직접 ai한테넘기고 그걸로 서버 연동하는데
뚝딱뚝딱해서 신기하긴하지만 보안이슈는 없나 싶음(당장 영상에 영상제작자 분의 프로젝트 API 키가 노출됨 ... )

### 피그마 메이크 -> 웹용 우리 범위가 아님 ❌
- 애니메이션 이런거 처리잘함(영상의 예시 참고)
- 실질적으로 피그마에서 어떤 컴포넌트 같은거 부분부분 만들기 좋음(사실상 개발자 cli 툴 느낌)
- 디자이너의 손이 필요해보임
- 디자인 시안 목업 따는데에 특화된거로 보임 -> 목업에서 인터렉션이 죄다 구현됨
- 피그마 기반이라 그런지 피그마를 이용해서 실제 UI 부분 코드화 시키는것이 더 중점적인 기능으로 보임 

### lovable -> 웹용 우리 범위가 아님 ❌
- 피그마 메이크랑 비슷해보임
- 외부 기능과 연동이 쉽다고 함(mcp 지원이 잘됨)
- 무료로 하루에 5개의 요청만 가능
- 솔직히 개발자 입장에서 피그마 메이크나 이거나 똒같음

### 프레이머 -> 웹용 우리 범위가 아님 ❌
- 와이어 프레임을 뽑는 기능을 제공함(타 ai는 이걸 주요한 기능으로 밀지 않음)
- 뭐 딱히 위력적이여 보이지는 않음

### readdy ‼️ -> 웹용 우리 범위가 아님 ❌
- 실제 프로덕트를 만드는데 유용해보임 -> 다음 기능을 생성하는 자연스러운 단계가 제공됨
- 기획자와 디자이너를 끼고 뭘 만드는 느낌을 살짝 첨가한 그런 툴?
- 서버 연동에대한 사항들도 ai 개발단부터 고려하려는것이 보임(간단하게나마 테이블 설계까지함)
- 초심자가 ai로 프로덕트를 만드는 시작점으로 좋아보임
- ai 영상까지 만들수도 있음

결론: ai 죄다 웹 전용으로 잡혀있는 경우가 많다. 앱은 뭔가 소외당하고 있는것 같다.
