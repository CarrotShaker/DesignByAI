# 1️⃣ UXPilot

🔗 https://uxpilot.ai/a/ui-design?page=4AWTxyT1vWKkhprvK6l8  

## 📌 요청 프롬프트
안드로이드 당근마켓의 홈화면의 바텀 네비게이션과  
홈 화면의 상품 리스트,  
상품 클릭 시 열릴 상품 상세 페이지를  
클론코딩하기 위한 디자인 시안 생성

---

## ✅ 주요 기능

### 1. 와이어프레임 vs 색채 포함 완성형 디자인 선택 가능

- 저충실도 Wireframe 선택 가능
- 컬러 및 스타일 포함된 완성형 UI 선택 가능

### 2. 디자인 시스템 선택 가능

- Material / iOS 스타일 등 선택 가능
- 디자인 톤 조절 가능

### 3. 화면별 프롬프트 생성 + UX 플로우 생성 가능

- 화면 단위 프롬프트 자동 생성
- 화면 간 동작 플로우 설계 가능
- UX 흐름 중심 설계 지원

---

## ✅ 장점

- Figma export 가능 (※ 결제 필요)
- 소스코드 보기 가능 (※ 결제 필요)
- UX 플로우까지 자동 설계 가능
- 디자인 시스템 선택 가능

---

## ❌ 단점

- 원하는 디자인이 즉시 나오지 않음
- 하나의 화면 단위로 프롬프트를 나누어 작성해야 했을 가능성 있음
- 당근마켓 레퍼런스를 완전히 따르지 않음
- Figma 대비 즉각적인 결과물 만족도는 낮았음

# 2️⃣ Figma (AI 기반 코드 생성 활용)

## ✅ 장점

- TypeScript 기반 코드 자동 생성
- CSS 포함 코드 확인 가능
- 컴포넌트 제약을 코드로 확인 가능

예:
- 특정 이미지가 화면 하단 기준 몇 dp 위에 배치되었는지 확인 가능
- 오토레이아웃 제약 조건 확인 가능

- Figma 연동 가능 (※ 결제 필요)
- 소스코드(TypeScript + CSS) 확인 가능

---

## ❌ 단점

- 디자이너 관점에서 웹 배포 + 코딩 도우미 느낌
- 개발자 관점에서 UI/UX를 자동 설계해주는 느낌은 약함
- UX 플로우 고려는 UXPilot 대비 부족한 느낌

# 🧠 결론

- UX 설계 중심 접근 → UXPilot이 유리
- 구체적 레이아웃 제약 및 코드 기반 확인 → Figma가 유리
- 클론코딩 관점에서는 프롬프트 세분화가 중요







2. 피그마

장점: 타임스크립트 기반으로 코딩을 해줌, 컴포넌트 제약이 언어로 선언되어 있기 때문에 디자인 시안으로만 확인할 수 없던 컴포넌트의 제약을 확인할 수 있음(ex. 특정 사진이 화면 하단을 기준으로 몇 dp 위에 올라와있는지)
피그마 연동 가능하며 결제 필요, 소스코드(타입스크립트 + css) 확인 가능


단점: 디자이너 입장에서 코딩 + 웹배포를 도와주는 느낌, 개발자 입장에서 ui/ux를 만들어주는 느낌은 아님
1번과 비교했을 때 ux까지 고려되진 못한 느낌은 있음

답변1.
당근마켓 스타일의 홈화면을 만들어드리겠습니다. 바텀 네비게이션, 상품 리스트, 상품 상세 페이지를 구현하겠습니다.

답변2. 
당근마켓 스타일의 앱을 완성했습니다! 하단 네비게이션으로 여러 화면을 탐색할 수 있고, 홈 화면에서 상품을 클릭하면 상세 페이지로 이동합니다. 당근마켓의 시그니처 오렌지 컬러(#FF6F0F)를 사용했으며, 실제 앱과 유사한 레이아웃과 인터랙션을 구현했습니다.

주요 기능:

바텀 네비게이션: 홈, 동네생활, 내 근처, 채팅, 나의 당근 5개 탭
홈 화면: 상품 리스트 카드 형태로 표시 (이미지, 제목, 위치, 가격, 좋아요/채팅 수)
상품 상세: 이미지, 판매자 정보, 상품 설명, 관련 상품, 하단 고정 버튼(좋아요/채팅하기)
